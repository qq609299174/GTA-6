# Core Loop

## Overview

The core loop is the foundation of Tang Frontier.

The game is built around repeated day-night cycles where players develop a town, prepare for danger, survive supernatural attacks, and use the aftermath to expand into new regions.

```text
Build and prepare -> Defend and react -> Recover and upgrade -> Explore and unlock
```

The player should always feel that each phase affects the next one.

- Poor preparation makes night harder.
- Successful night defense protects resources and morale.
- Exploration unlocks tools that change future defenses.
- Damaged buildings create long-term consequences.
- Good economy and logistics make the town stronger over time.

## Day Phase

The day phase is about planning, growth, and opportunity cost.

Players choose how to spend limited time before the next attack.

Common actions:

- Gather wood, stone, ore, herbs, food, and ritual materials
- Build new structures
- Upgrade existing buildings
- Repair night damage
- Assign NPC workers
- Craft equipment and supplies
- Manage marketplace inventory
- Fulfill trade or town orders
- Research enemy weaknesses
- Accept main and side quests
- Explore nearby areas
- Prepare traps and defensive layouts

The day phase should not feel like downtime. It should create meaningful strategic pressure.

Example decisions:

- Upgrade the gate or build the marketplace?
- Send players outside for ore or keep them repairing walls?
- Use iron for weapons or defensive spikes?
- Sell medicine for coins or save it for night?
- Research tonight's enemy type or unlock a new building blueprint?

## Dusk Phase

Dusk is the commitment phase before danger begins.

Players review the town state and make final strategic choices.

Possible decisions:

- Which gate or district receives reinforcement?
- Which traps are armed?
- Which town ability is prepared?
- Which NPCs are assigned to guard duty?
- Which players defend town?
- Which players leave for a night expedition?
- Which resources are reserved for emergency repairs?
- Whether to trigger a high-risk, high-reward night modifier

Dusk should feel like the moment when plans become commitments.

## Night Phase

Night is the pressure phase.

Enemies attack the town through multiple routes and systems.

The design goal is that night defense is not only combat. It is crisis management under pressure.

Possible threats:

- Main gate assault
- Side route infiltration
- Underground breach
- Rooftop enemy entry
- Siege unit targeting buildings
- Ritual enemy attacking seal points
- Civilian panic event
- Building fire or corruption event
- Enemy commander buffing waves
- Boss phase that changes town layout pressure

Player actions during night:

- Fight enemies
- Repair gates and buildings
- Operate traps and siege tools
- Heal allies and NPCs
- Move supplies between buildings
- Maintain rituals or seals
- Rescue civilians
- Destroy enemy objectives outside the walls
- Hold different lanes at the same time

## Outside Objectives During Night

Night should support split-team gameplay.

Some players may leave the town to complete dangerous objectives while others defend.

Examples:

- Destroy an enemy altar to weaken the final wave
- Close a breach before ghost enemies enter town
- Rescue trapped civilians before they are converted
- Kill a commander before he reaches the gate
- Recover a ritual item needed to activate the town barrier
- Burn siege supplies before they are used against the town

This creates strategic tension:

- Keeping everyone inside is safer immediately.
- Sending players outside is riskier but may reduce long-term pressure.

## Aftermath Phase

After night ends, the town resolves consequences.

Possible rewards:

- Materials
- Coins
- Reputation
- Building blueprints
- Research fragments
- NPC survivors
- Quest clues
- Rare enemy drops
- New region access

Possible damage:

- Broken gates
- Damaged buildings
- Injured NPCs
- Lost supplies
- Lower morale
- Higher corruption
- Destroyed traps
- Unfinished objectives causing future threats

The aftermath phase should make the night feel consequential.

## Multi-Cycle Progression

Each day-night cycle should move the game forward.

A cycle can change:

- Town layout
- Available buildings
- Worker population
- Enemy types
- Economy strength
- Corruption level
- Region unlocks
- Quest state
- Player builds
- Defense strategy

The core loop should avoid feeling like identical repeated waves. Each cycle should introduce new pressure, new tools, or new consequences.

## Example Early Cycle

### Day 1

- Gather wood and stone
- Repair Town Hall
- Build basic gate
- Build Blacksmith foundation

### Night 1

- Small enemy wave attacks the gate
- Players learn combat and repair basics
- One weak elite enemy appears

### Aftermath 1

- Unlock Workshop
- Receive first clue about supernatural activity
- Gate needs repair

### Day 2

- Build Workshop
- Craft spike traps
- Send one player outside to gather ore
- Prepare stronger defense

### Night 2

- Enemies attack gate and side route
- Siege enemy damages Workshop
- Optional outside objective can reduce final wave

### Aftermath 2

- Unlock Marketplace
- Receive first Qin-related clue
- Town morale changes based on defense success

## Design Success Criteria

The core loop succeeds if:

- Day decisions clearly affect night survival
- Night damage clearly affects the next day
- Exploration changes future defense options
- Buildings feel useful during and outside combat
- Players naturally divide responsibilities
- The town feels worth protecting
- The player wants to do one more cycle
